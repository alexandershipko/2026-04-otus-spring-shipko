Требования к реализации:

0. Учесть актуальные требования прошлой работы
1. Перенести все свойства в application.yml
2. MesageSource должен быть сконфигурирован Spring Boot автоматически, на основе настроек из application.yml
3. Локализовать выводимые сообщения и вопросы с вариантами ответов
4. Сделать собственный баннер для приложения +
5. Перенести и адаптировать тесты
6. Вместо отдельных зависимостей в файле сборки, для тестирования использовать spring-bootstarter-test
7. В остальном, требования к тестам не меняются
8. Опционально: Использовать ANSI-цвета для баннера +
9. Заготовка для выполнения работы: https://github.com/OtusTeam/Spring/tree/master/templates/hw03-spring-boot
10. Задание сдаётся только после приема предыдущей работы
11. Задание сдаётся в виде ссылки на pull-request в чат с преподавателем в личном кабинете ОТУС, не в Telegram
12. В pull-request должно присутствовать только то что касается текущей работы. Временные файлы и файлы IDE не должны попадать в PR.
13. Инструкция по работе с pull-request-ами находится в документе "Настройка окружения.pdf" в материалах занятия.
14. Вопросы можно задавать в чате ДЗ, но для оперативности рекомендуем делать это в группе Telegram.
15. Код, написанный в данном ДЗ будет использоваться в дальнейших домашних заданиях первого блока курса
