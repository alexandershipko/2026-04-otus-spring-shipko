Требования к реализации:

Создать приложение, каталог книг в библиотеке

0. Использовать Spring JDBC и реляционную базу H2
1. Предусмотреть сущности авторов, книг и жанров. Каждая должна храниться в своей таблице
2. Предполагаются отношения многие-к-одному (у книги один автор, но у автора может быть несколько книг, то же касается книг и жанров)
3. Интерфейс выполняется на Spring Shell (CRUD книги плюс, как минимум, операции вывода всех авторов и жанров)
4. Создание и инициализация схемы БД должно происходить через schema.sql + data.sql или через систему миграций (Liquibase/Flyway)
5. С помощью @JdbcTest сделать интеграционные тесты всех методов дао книг (со встроенной БД) !!!

Дополнительные требования

0. Проблема N+1 должна быть решена
1. Использовать NamedParametersJdbcTemplate
2. Не делать абстрактных или обобщенных Dao
3. Не делать абстрактных или обобщенных сущностей
4. Не делать двунаправленных связей (в книге автор, в авторе книги)
5. Без фанатизма!
6. Опциональное усложнение - отношения многие-ко-многим для одной сущности!!!
   
Опционально

0. Создание и инициализация схемы БД сделать через систему миграций (Liquibase/Flyway) - сделать в 6 hw!
1. Отношение книга-жанр, многие-ко-многим (должен использоваться соответствующий вариант заготовки)
