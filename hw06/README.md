Требования к реализации:

0. Использовать Spring ORM, JPA и Hibernate (EntityManager) в качестве технологии доступа к данным +
1. Spring Data пока использовать нельзя +
2. Добавить сущность "комментария к книге" (много комментариев к одной книге) !
3. Реализовать CRUD операции для новой сущности. Получение всех комментариев делать не нужно.
   Только конкретного комментария по id и всех комментариев по конкретной книге по ее id !
4. Загрузка связей сущностей не должна приводить к большому количеству запросов к БД 
   или избыточному по объему набору данных (проблема N+1 и проблема произведения таблиц) +
5. DDL через Hibernate должно быть отключено +
6. LAZY-связи не должны присутствовать в equals/hashCode/toString. В т.ч. за счет @Data +
7. Проблема N+1 должна быть решена теми же способами, что и в прошлой работе,
   но уже через инструменты, предоставляемыми ORM
8. Аннотация @Transactional должна присутствовать только на методах сервиса +


9. Покрыть все репозитории тестами, используя H2 базу данных и @DataJpaTest !
10. В тестах репозиториев, для проверки или подготовки данных использовать TestEntityManager
    (только методы не требующие написания запросов). Сами репозитории для этого использовать нельзя !
11. Написать интеграционные тесты сервисов книг и комментариев, где проверить, что доступ к связям, 
    которые используются снаружи серивсов не вызывают LazyInitialzationException
12. Не забыть учесть в тестах, что тестовый контекст кэшируется, т.е. должны проходить как все тесты разом,
    так и по отдельности !
13. Для приема работы тесты из заготовки должны проходить +
